# Template Setup Guide

## Quick Start

1. **Browse Templates**: Visit [TemplateHub](../index.html) to explore available templates
2. **Download**: Click the download button on any template
3. **Extract**: Unzip the downloaded file
4. **Open**: Open `index.html` in your favorite code editor
5. **Customize**: Modify the content, colors, and styling to match your needs

## Template Structure

Each template includes:
- `index.html` - Main HTML file
- `css/` - Stylesheet directory
- `js/` - JavaScript files (if applicable)
- `images/` - Image assets
- `README.md` - Template-specific documentation

## Customization

### Colors
Most templates use CSS custom properties (variables) for easy color customization:

```css
:root {
  --primary-color: #667eea;
  --secondary-color: #764ba2;
  --accent-color: #f093fb;
}
```

### Fonts
Templates use Google Fonts. You can change fonts by:
1. Updating the Google Fonts link in the HTML head
2. Modifying the CSS font-family declarations

### Images
Replace placeholder images with your own:
1. Add your images to the `images/` folder
2. Update image paths in HTML and CSS files

## Integration

### With Frameworks
Templates can be integrated with popular frameworks:
- **React**: Convert HTML to JSX components
- **Vue**: Use as template in Vue components
- **Angular**: Integrate into Angular components

### With CMS
Most templates work with:
- WordPress (convert to PHP templates)
- Jekyll (for static sites)
- Ghost (blog templates)

## Best Practices

1. **Responsive Design**: All templates are mobile-first
2. **Performance**: Optimize images and minify CSS/JS for production
3. **Accessibility**: Templates include ARIA labels and semantic HTML
4. **SEO**: Update meta tags and structured data
